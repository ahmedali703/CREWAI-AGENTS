    # API Reference 🔌

This section provides details on all available API endpoints.

---

## 1️⃣ Search for Products

### **Endpoint**

    POST /api/search

    
### **Description**
🔹 This endpoint searches for a product and generates a procurement report.

---

### **Request Body**
```json
{
  "productName": "Laptop",
  "country": "USA",
  "resultCount": 5
}
    Parameter Type Description Required
    productName String The name of the product to search for. ✅ Yes
    country String The target country for the search. ✅ Yes
    resultCount Integer The number of product results required. ✅ Yes

    Response Example

    {
  "status": "success",
  "message": "Report generated successfully",
  "report_url": "/reports/procurement_report.html"
}
    Parameter Type Description
    status String Indicates if the request was successful.
    message String A message describing the result.
    report_url String The URL to access the generated report.

    2️⃣ Retrieve a Procurement Report

    Endpoint

    GET /reports/{filename}

    Description

    🔹 Retrieves a generated procurement report by filename.

    Request Parameters

    Parameter Type Description Required
    filename String The name of the report file to retrieve. ✅ Yes

    Response

    • Returns the HTML file of the generated procurement report.

    ✅ Notes

    • The /api/search request must be completed successfully before retrieving a report.
    • The report URL will be provided in the search response.

    Now you're all set to interact with the API! 🚀 ```
